# Classroom Sprint 2B.1 使用说明

## 第一次连接本地文件夹

1. 使用桌面版 Google Chrome 打开 Classroom。
2. 进入左侧 **Storage & backup**。
3. 点击 **Connect Classroom Data folder**。
4. 在文件选择窗口中选择你已经建立好的 `Classroom Data` 文件夹。
5. Chrome 请求权限时选择允许。

Classroom 会检查并补齐以下目录：

- Imports
- Resource Files
- Backups
- Backups/Recovery
- Exports
- Archive

## 第一次连接时如何选择

- 文件夹中没有 Classroom workspace：系统会把当前浏览器数据保存进去。
- 浏览器与文件夹内容一致：显示 Folder synced。
- 两边内容不同：系统不会自动覆盖，会要求选择 **Use folder copy** 或 **Keep browser copy**。

## 自动保存

开启 **Auto-save to connected folder** 后，Planner、Library、Import Center 等发生变化时，会在短暂延迟后写入：

`Classroom Data/Classroom-Workspace.classroom.json`

覆盖前，旧版本会保存在：

`Classroom Data/Backups/Recovery/Classroom-Previous.classroom.json`

系统还会每隔至少 15 分钟保留一个时间戳恢复点，最多保留 20 个自动恢复文件。

## 手动备份

点击 **Backup now**，系统会在 `Backups` 中建立一个带日期和时间的备份。手动备份不会被自动删除。

## 注意

- 不要把 `Classroom Data` 上传到公开 GitHub。
- 不要把 Google Drive 文件夹设置为 Anyone with the link。
- 当前备份是可阅读的 JSON 文件；包含学生信息时必须保持文件夹私密。
- 不要清除浏览器网站数据，除非已确认文件夹备份可恢复。


## Sprint 3：全局搜索

1. 点击左侧 **Search**，或在 Chrome 中按 **Command + K**（Windows 使用 **Ctrl + K**）。
2. 搜索课程标题、学习者、主题、标签、教学记录、备注、资源名称或导入文件名。
3. 可以按照资料类别、日期范围、学习者和标签继续筛选。
4. 点击 **Save current search** 保存常用筛选；保存的搜索会随 Classroom 工作区一起备份。
5. 课程结果可直接在周计划中定位或编辑；Library 结果可直接打开链接或编辑；其他记录可查看完整详情。
6. 搜索索引只在当前浏览器生成，不会上传到 GitHub。
